Data Overview 3
Data Sources 
The primary data source for this project is the National Vital Statistics System, which provides detailed stroke mortality statistics for U.S. adults over 35. This dataset, available on Kaggle, includes 3-year averages and age-standardized mortality rates, segmented by state, county, gender, and ethnicity. These comprehensive data points are essential for identifying demographic and geographic disparities in stroke outcomes.


System Architecture 
The architecture integrates data ingestion, preprocessing, and machine learning workflows to enable a streamlined analysis pipeline. Starting from raw data in CSV format, data cleaning and preparation are conducted to ensure accuracy and consistency. The project uses NexusMax for model building, allowing advanced predictive analysis on stroke mortality trends. Data unification and preparation are handled within Jupyter and Python, ensuring compatibility across processes. Finally, the cleaned and structured data is loaded into a Data Warehouse built with SSIS, providing a reliable storage solution for long-term data analysis and supporting public health insights.


Data Features 
The dataset includes crucial indicators for stroke mortality, providing insights across various demographics and locations. Key features comprise location identifiers (state, county), demographic details (gender, race/ethnicity), and mortality rates with 3-year averages and age-standardized values. This level of detail supports meaningful KPI calculations, such as stroke mortality rate by gender and race/ethnicity, and highlights geographic hotspots with elevated mortality rates. By examining these data features, the project identifies patterns crucial for targeting public health interventions and reducing stroke-related deaths.
